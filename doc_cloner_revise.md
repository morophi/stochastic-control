# Document Cloner — Operational Rules (Revised)

## 목적

이 문서는 시스템 운용을 설명하지 않는다.  
이 문서는 **사고 운용 방식**을 정의한다.

---

## 기본 사용 흐름

1. AI 출력 수신
2. 출력 분리 (사실 / 추론 / 가정)
3. 사용자 판단 개입
4. 채택 또는 거부
5. 판단 근거 기록

---

## Mode A / Mode B

- Mode A: 탐색 모드 (아이디어 확장)
- Mode B: 검증 모드 (판단 제한)

모드는 사용자가 명시적으로 선택한다.

---

## 기록 원칙

- 판단 근거는 간단해도 된다
- 기록하지 않은 판단은 존재하지 않는 것으로 간주한다

---

## 교육 환경 사용 고지

본 규약은 교육 환경에서도 적용된다.

그러나:

- 학습은 책임을 면제하지 않는다
- 실험은 결과를 정당화하지 않는다

학생은 다음을 수행해야 한다.

- AI 출력 채택 이유 설명
- 거부 판단 근거 명시
- 사고 경로 기록

교육은 책임 훈련이다.

---

## 중단 조건

다음 상황에서는 사용을 중단한다.

- 출력 결과만 신경 쓰게 될 때
- 판단 설명이 불가능할 때
- AI를 권위로 느끼기 시작할 때

이 프레임워크는 준비되지 않은 사용을 허용하지 않는다.
